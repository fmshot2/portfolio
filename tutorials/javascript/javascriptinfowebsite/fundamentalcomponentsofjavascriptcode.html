Fundamental Components of Javascript Code

Follow Us

Let's take a look at the fundamental components of javascript code.

Statements are syntax constructs that perform actions or commands. We can have multiple statements in our code, separated by semicolons. For instance, we can split "Hello World" into two alerts, like this:

alert('Hello'); alert('World');

Usually, statements are written on separate lines for better readability:

alert('Hello');

alert('World');

Where can JavaScript Be Used?

JavaScript can be run in the browser, in servers, or on any device that has a "JavaScript engine".

In most cases, a semicolon can be omitted when a line break exists, as JavaScript interprets the line break as an "implicit" semicolon. This is known as automatic semicolon insertion. However, there are cases where a newline doesn't mean a semicolon, which can lead to hard-to-find errors. For instance:

alert(3 + );

1;

* 2;

Here, JavaScript does not insert semicolons because the line ending with a plus "+" is an "incomplete expression," and adding a semicolon there would be incorrect.

While it's possible to leave out semicolons most of the time, we recommend putting them between statements separated by newlines. This is a widely adopted rule by the community and helps avoid errors, especially for beginners.

In-browser JavaScript can do everything related to webpage manipulation, interaction with the user, and the webserver.

Comments

Comments can be added to scripts to describe what the code does and why. They can be put anywhere in the code and do not affect execution because the engine simply ignores them. One-line comments start with two forward slash characters "//," and the rest of the line is a comment. Multiline comments start with a forward slash and an asterisk "/" and end with an asterisk and a forward slash "/." The content of comments is ignored, so if we put code inside /* ... */, it won't execute. Sometimes, it's useful to disable a part of code temporarily by commenting it out like this:

/* Commenting out the code

alert('Hello'); */

alert('World');

Use Hotkeys!

In most editors, a line of code can be commented out by pressing the Ctrl+/ hotkey for a single-line comment and something like Ctrl+Shift+/ – for multiline comments (select a piece of code and press the hotkey). For Mac, try Cmd instead of Ctrl and Option instead of Shift.
